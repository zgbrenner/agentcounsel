# OSS Borrowing Map

This map records how AgentCounsel should learn from the listed open-source
repositories without copying unsafe or incompatible material. Licenses were
checked against GitHub repository metadata on 2026-05-28, with a second pass
for repositories that GitHub reported as `NOASSERTION` or no license file.

AgentCounsel is Apache-2.0. Direct copying or close adaptation should be
limited to license-compatible sources and must follow
`docs/THIRD_PARTY_ATTRIBUTION_POLICY.md`. This PR copies no external code,
prose, prompts, schemas, examples, or eval content, so `NOTICE` does not need
an update.

## Direct-Reuse Shortlist

The safest near-term sources for direct pattern adaptation are permissively
licensed and aligned with existing AgentCounsel systems:

- `anthropics/knowledge-work-plugins` for platform packaging.
- `hardikpandya/stop-slop` for a legal prose quality pass, rewritten in
  AgentCounsel's own words.
- `harveyai/harvey-labs` for eval maturity.
- `public-api-lists/public-api-lists` for connector registry organization.
- `mukul975/Anthropic-Cybersecurity-Skills` for large skill-registry patterns.

## Inspiration-Only Watchlist

These projects may guide product thinking but should not be copied under the
current license strategy:

- GPL/AGPL projects: `p-e-w/heretic`, `tinyhumansai/openhuman`,
  `frappe/erpnext`, `willchen96/mike`.
- No-license projects: `iii-hq/iii`, `PolsiaAI/Polsia`, `cursor/plugins`,
  `veronica-builds/claude-for-legal-web`, `browserbase/skills`,
  `ComposioHQ/awesome-claude-skills`.
- Custom/share-alike projects: `twentyhq/twenty`, `multica-ai/multica`,
  `awesome-selfhosted/awesome-selfhosted`.
