## Stop and Ask — Missing Requirement Source

The user named a regulatory topic but did not provide the text of the regulation, framework, or standard. Requirements cannot be constructed from background knowledge alone. Building a gap matrix without a traceable source would amount to inventing requirements, which the skill will not do.

## What This Skill Requires

- The text of the regulation, framework, or standard to be assessed (or stable links to a verified copy).
- A description of the entity's current controls, policies, and systems sufficient to compare against the requirement provisions.

## What This Skill Will Not Do

- Will not build a requirements list from the model's background knowledge of the topic.
- Will not produce a gap matrix with no traceable source.
- Will not assign severity to gaps measured against invented requirements.
- Will not present any compliance conclusion.

## Follow-Up the Reviewer May Need

If the user's description of current controls is too vague to compare against the (yet-to-be-provided) requirement source, targeted follow-up questions will be needed once the source is supplied. Examples:

- Which internal program area owns the activity described?
- Is the control documented and, if so, where?
- What cadence and ownership are assigned to monitoring?
